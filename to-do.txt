1. HIGHEST-PRIORITY GLOBAL FIXES
=====================================================================

1.1 Replace notebook-style screenshots with presentation exports
- Do not paste raw notebook screenshots directly into slides when avoidable.
- Re-export each final graph from matplotlib at high resolution (minimum 300 dpi; 400 dpi preferable if staying raster).
- Prefer tight bounding boxes so the plot area fills more of the slide.
- If possible, export vector versions (PDF/SVG) first, then place high-resolution renders into PowerPoint.
- Avoid the look of “plot screenshot dropped onto black slide”; either:
  - place each graph inside a white panel/card with even margins, or
  - use a consistent white rectangular background behind all plots.

1.2 Standardize graph typography across the whole deck
- Use one consistent font size scale for all plots.
- On-slide readability target:
  - title: large enough to read from the back of the room,
  - axis labels: clearly legible,
  - tick labels: not tiny,
  - legend text: no smaller than the tick labels,
  - stats box text: only include if still readable when the graph is placed on the slide.
- Increase font sizes before export rather than enlarging low-resolution images inside PowerPoint.

1.3 Simplify statistics boxes
- Current fit boxes often contain too many significant figures and too much regression detail.
- For presentation slides, keep only the quantities that help the audience:
  - slope (rounded sensibly),
  - R^2 (if useful),
  - final wavelength estimate with uncertainty.
- Remove over-precise slopes/intercepts such as 6.8561e-06 unless that exact precision is scientifically necessary.
- Round all reported values to sensible significant figures tied to the uncertainty.

1.4 Make axis labels human-readable
- Avoid notebook-internal or workbook-internal labels.
- Avoid labels that only make sense to the person who wrote the spreadsheet.
- Replace abstract/internal names like:
  - “combined scaled”
  - “workbook transform”
  - “workbook-style”
  - “index n”
  with physically meaningful language.
- Every axis should tell the audience exactly what is plotted and in what units.

1.5 Avoid scientific-notation offsets unless essential
- Several plots use automatic axis scaling like 1e-5 or 1e7.
- For a presentation, convert axes to friendlier units where possible so the axis does not rely on a tiny scientific-notation offset in the corner.
- Example: instead of leaving aΔy in raw SI units with a 1e-5 offset, convert to a more presentation-friendly scaled unit or explicitly label the axis as “aΔy (×10^-5 m^2)”.

1.6 Use consistent naming across slides and plots
- Use the same terminology everywhere for the same method.
- In particular, use one final naming scheme for:
  - single-slit method,
  - grating maxima method,
  - grating missing-order/envelope method,
  - Fresnel half-period zones,
  - straight-edge Fresnel diffraction.
- Do not let notebook plot titles and slide bullet text use different names for the same analysis.

1.7 Clarify what the audience should conclude from each graph
- Each graph should have a one-line takeaway either in the title, subtitle, or nearby slide text.
- Good structure for each result slide:
  - what was plotted,
  - why it should be linear,
  - what wavelength came out,
  - what the main weakness/error source was.
